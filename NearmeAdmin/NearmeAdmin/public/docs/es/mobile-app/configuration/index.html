Configuración

Configuración global

Abre el archivo app.config.ts localizado en src/app y ajusta el valor de las siguientes variables:

  • SERVER_URL - La URL del servidor. Debe coincidir con el valor declarado en el Dockerfile.

  • APP_ID - El ID de la aplicación. Debe coincidir con el valor declarado en el Dockerfile.

  • GOOGLE_MAPS_API_KEY - La llave para la API de Google Maps. Requerida para que el mapa funcione correctamente. Puedes usar la misma llave generada durante la configuración del portal web ó crear una nueva desde aquí.

  • BANNER_ID - ID del banner de AdMob. Opcional. Dejalo vacío si no necesitas mostrar publicidad de Google. En caso contrario ve al portal de AdMob, crea una nueva app y crea un nuevo ID para el banner.

  • TRACKING_ID - Código de seguimiento de Google Analytics. Opcional. Dejalo vacío si no deseas medir las estadísticas de uso.

  • HEADER_COLOR - Color para el status bar (sólo Android). Opcional. Cualquier valor hexadecimal es válido.

  • DEFAULT_UNIT - Unidad de distancia predeterminada. Valores posibles: km, mi.

  • DEFAULT_LANG - Lenguaje predeterminado. Valores posibles: en, es. Si deseas agregar otro lenguaje crea un nuevo archivo en la carpeta src/assets/i18n. Luego copia y pega el contenido del archivo en.json dentro del nuevo archivo y edita los textos libremente.

Ajustar metadatos

Antes de publicar nuestra aplicación en las tiendas es necesario actualizar algunos datos declarados en el config.xml. Actualiza el ID del paquete y el nombre cómo se muestra a continuación.

<widget id="com.yourdomain.appname" version="1.0.0">
    <name>Nombre de tu app</name>
</widget>

Actualiza también la información que se encuentra dentro de la etiqueta </author>.

Configurar Notificaciones Push

Abre el archivo config.xml y actualiza el valor de ParseServerUrl y ParseGcmSenderId.

Ejemplo:

<!-- Ambos valores deben coincidir con el Dockerfile -->
<preference name="ParseServerUrl" value="https://nearme.quanlabs.com/parse/" />
<preference name="ParseGcmSenderId" value="XXXXXXXXXXX" />

Es importante que la URL tenga al final el caracter / para que las notificaciones funcionen correctamente.

Configurar Facebook Login

En el mismo config.xml edita las variables APP_NAME y APP_ID del plugin de Facebook.

<plugin name="cordova-plugin-facebook4" spec="^2.1.0">
    <variable name="APP_ID" value="YOUR_APP_ID" />
    <variable name="APP_NAME" value="YOUR_APP_NAME" />
    <variable name="ANDROID_SDK_VERSION" value="4.+" />
</plugin>

Para más información sobre cómo generar estos valores visita la documentación del plugin en Ionic Native.

Personalizar colores

Cambiar los colores es tan fácil como editar el mapa de colores $colors en el archivo variables.scss.

$colors: (
    primary:    #e91e63,
    secondary:  #f4f4f4,
    danger:     #f53d3d,
    light:      #f4f4f4,
    accent:     #ccc,
    dark:       #7e7e7e,
    darker:       #444,
    fb:         #3b5998,
);

Para ajustar el color del toolbar edita las variables $gradient-bg y $secondary-bg dentro del app.scss.

Actualizar Icono y SplashScreen

Reemplaza los archivos localizados en la carpeta /resources y en src/assets/img.

Para más información consulta la documentación oficial de Ionic.
